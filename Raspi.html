1. LED simple

Ánodo (+)GPIO + resistencia 330Ω

PWM: Para variar brillo usa GPIO 12, 13, 18 o 19 — tienen PWM hardware. El resto solo on/off (0/1).

3. Sensor Ultrasonidos (HC-SR04)

GPIO

Conexión

PinDestino
Vcc5V Raspi
TrigGPIO24 (salida)
EchoGPIO25 — con divisor de tensión
GNDGND Raspi
¡Ojo! Echo devuelve 5V pero el GPIO aguanta 3.3V máx. Divisor resistivo: R1=1kΩ (Echo→Vout), R2=2kΩ (Vout→GND). Vout = 5V × 2/(1+2) = 3.33V ✓
Funcionamiento: envías pulso de 10µs por Trig → mides duración del Echo → distancia = tiempo × 343 / 2 (m/s del sonido).

4. Pulsador

GPIO

Pull-up (más común)

PinDestino
Un ladoGPIO + R 10kΩ → 3.3V
Otro ladoGND
Pull-up interno: La Raspi tiene resistencias pull-up internas activables por software (GPIO.setup(pin, GPIO.IN, pull_up_down=GPIO.PUD_UP)). No hace falta resistencia externa.
Flancos: con GPIO.add_event_detect(pin, GPIO.FALLING) se detecta el momento del pulso sin polling.

5. Acelerómetro (MPU-6050)

I2C

Conexión

Pin MPUDestino
VCC3.3V Raspi
GNDGND
SCLGPIO3 (SCL)
SDAGPIO2 (SDA)
INTGPIO cualquiera (flancos)
AD0GND → dir. 0x68 / VCC → 0x69
Display: si el display también es I2C, se conectan SCL y SDA al mismo bus — cada dispositivo tiene dirección distinta. No hace falta bus separado.
Flancos: el pin INT genera una interrupción cuando hay nueva medida. Conectarlo a un GPIO y usar GPIO.RISING para no hacer polling.

6. Sensor Luminosidad (LDR + condensador)

GPIO

Conexión

ComponenteConexión
LDR extremo 13.3V
LDR extremo 2Nodo central → GPIO
Condensador +Nodo central
Condensador −GND
Principio: La Raspi no tiene ADC. Se mide el tiempo que tarda el condensador en cargarse (RC). Como LDR varía su resistencia con la luz, el tiempo de carga varía — así se obtiene valor analógico de luminosidad.
Condensador típico: 1µF. Descarga el GPIO a GND, luego pon el pin como entrada y mide cuánto tarda en subir a HIGH.

7. Sensor CO₂ (UART)

UART pila

Conexión

Pin SensorDestino

GNDGND común
TXDGPIO15 (RX de la Raspi)
RXDGPIO14 (TX de la Raspi)
3.3V3.3V Raspi (ref. lógica)
Regla UART: TX del sensor → RX de la Raspi y viceversa. "El que habla (TX) conecta al que escucha (RX)."
Pila: necesita alimentación externa porque su consumo supera los límites de la Raspi.

11. Display OLED (SSD1306)

I2C

Conexión

Pin DisplayDestino
VCC3.3V Raspi
GNDGND
SCLGPIO3 (SCL) — mismo bus I2C
SDAGPIO2 (SDA) — mismo bus I2C
Bus compartido: El display y el acelerómetro comparten GPIO2/GPIO3. Dirección I2C del display suele ser 0x3C. Verificar con i2cdetect -y 1.
Librería Python: luma.oled o adafruit-circuitpython-ssd1306. El acelerómetro genera el dato y el display lo muestra — por eso el enunciado los vincula.

12. Servomotor (dirección)

PWM pila recomendado

Conexión

CableDestino
Marrón / NegroGND
Rojo5V pila (o 5V Raspi si es pequeño)
Naranja / AmarilloGPIO13, 19, 18 o 12 (PWM)
Control por PWM: señal de 50Hz (periodo 20ms). El ancho del pulso define el ángulo: 1ms = −90°, 1.5ms = 0° (centro), 2ms = +90°.
Para el coche: centro = recto, girar el pulso hacia 1ms o 2ms para girar las ruedas. Con gpiozero.Servo() es muy directo.
Pila: recomendable alimentación externa si hay más carga en la Raspi — un servo puede pedir 500–800mA en movimiento.
