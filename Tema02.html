01 — Qué es un sistema embebido

Sistema embebido (Embedded System)

Sistema de computación compuesto por hardware y software diseñado para realizar una o pocas funciones específicas dentro de un sistema más grande. A diferencia de una computadora de propósito general, está optimizado para una tarea concreta.

Características principales

  • › Función dedicada — no multifuncional
  • › Hardware y software "embebidos" dentro del dispositivo
  • › Diseño específico: consumo, tamaño, tiempo real, coste
  • › Microprocesador/microcontrolador + memoria + periféricos
  • › Pueden no tener SO de propósito general

Eficiencia — factores clave

  • › Eficiencia energética — prolongar batería
  • › Uso optimizado de recursos — mínima CPU y RAM
  • › Tiempo real — respuesta inmediata a eventos
  • › Tamaño compacto — hardware y software a medida
  • › Reducción de costes — producción en masa

Ley de Moore — relevancia en sistemas embebidos

La Ley de Moore dice que el número de transistores se duplica cada 2 años. No aplica directamente a sistemas embebidos porque su objetivo es la optimización y el ahorro de recursos, no la máxima potencia. Priorizan eficiencia, coste, tamaño y fiabilidad. Un chip menos potente pero más barato suele ser la elección correcta.

02 — Plataformas de desarrollo

Arduino vs Raspberry Pi vs ESP32

Arduino UNO

  • MCU: ATmega328 a 16 MHz
  • 14 pines digitales I/O (6 PWM)
  • 6 pines analógicos (ADC 10-bit)
  • Alimentación: USB o 7–20V DC
  • Lenguaje: C++ simplificado (IDE)
  • Sin SO — bare metal
Tiempo real Bajo consumo Sin red nativa Poca RAM

Raspberry Pi 3/4/5

  • SoC ARM Cortex-A (64 bits, 4 núcleos)
  • BCM2837B0 @ 1.4 GHz (Pi 3)
  • RAM: 512 MB – 8 GB LPDDR
  • 40 pines GPIO
  • Wi-Fi + Bluetooth + Ethernet
  • SO: Raspberry Pi OS (Linux)
  • Python, C/C++, etc.
Potente Linux completo Mayor consumo No tiempo real

ESP32

  • Dual-core LX6 de 32 bits
  • Wi-Fi 2.4 GHz + Bluetooth/BLE
  • Flash + SRAM interna
  • Múltiples GPIO, ADC, DAC
  • UART, I2C, SPI, I2S, PWM
  • IDE Arduino / MicroPython / ESP-IDF
  • Deep sleep para bajo consumo
Wi-Fi integrado Barato Sin Linux
Cuándo elegir cada uno
Arduino
Control en tiempo real, bajo consumo, sensor simple, sin red necesaria, coste limitado.
Raspberry Pi
Procesamiento complejo, visión por computador, IA, interfaz gráfica, servidor local, Python nativo.
ESP32
IoT con Wi-Fi/BT integrado, bajo consumo con conectividad, sensores en red, coste muy bajo.

ARM Cortex-A

Aplicaciones. Alto rendimiento. Ejecuta SO complejos (Android, Linux). Smartphones, tablets, servidores de bajo consumo. Unidades NEON + TrustZone.

ARM Cortex-R

Tiempo Real. Alta disponibilidad y fiabilidad. Automoción, almacenamiento, redes críticas. Pipelines largos, ejecución fuera de orden.

ARM Cortex-M

Microcontroladores / IoT. Coste y potencia críticos. Sensores, dispositivos médicos, control industrial. Instrucciones Thumb-2, memorias limitadas.

03 — Interfaz GPIO y protocolos de comunicación

Pines GPIO y sus tipos

GPIO (General Purpose Input/Output): pin digital programable como entrada (leer señales) o salida (enviar señales). No tiene un propósito fijo.

Tipo de pin Función Uso típico Notas importantes
Digital I/O Lee/escribe HIGH (3.3V/5V) o LOW (0V) LEDs, botones, relés Estado binario. Configurar antes de usar.
PWM Señal digital que simula analógica variando el ancho de pulso (duty cycle) Brillo LED, velocidad motor, servo Raspi: mejores pines PWM hardware → 12, 13, 18, 19
ADC Convierte señal analógica a digital Leer sensores analógicos (LDR, termistor) Raspberry Pi no tiene ADC integrado → necesita chip externo (MCP3008)
I2C (SDA/SCL) Bus serie síncrono, 2 cables, multi-esclavo Sensores BME280, acelerómetro, display Raspi: GPIO2=SDA, GPIO3=SCL. Pull-up a 3.3V incluido
SPI (MOSI/MISO/SCLK/CS) Bus serie síncrono, 4 cables, alta velocidad, full-duplex NFC, SD cards, ADC externo SPI0: CE0=GPIO8, CE1=GPIO7. NFC usa SPI + 5V
UART (TX/RX) Serie asíncrona, 2 cables (TX→RX del otro) Comunicación Raspi↔Arduino, GPS, módems Ambos deben tener el mismo baud rate. GND común obligatorio
Alimentación 3.3V, 5V, GND Alimentar periféricos GPIO Raspi opera a 3.3V. 5V solo en pines específicos de alimentación
I2C — 2 cables

Síncrono, maestro-esclavo, semidúplex. Múltiples dispositivos en el mismo bus (direccionamiento). Proceso: START → dirección → ACK → datos → STOP.

  • SDA — datos bidireccionales (GPIO 2)
  • SCL — reloj generado por maestro (GPIO 3)
  • VCC — 3.3V o 5V según periférico
  • GND — tierra común
SPI — 4 cables

Síncrono, full-duplex, más rápido que I2C. Cada esclavo necesita su propio pin CS. Sin direccionamiento — selección por hardware.

  • MOSI — Master Out Slave In
  • MISO — Master In Slave Out
  • SCLK — reloj del maestro
  • CE0/CE1 — Chip Select (GPIO8 / GPIO7)
UART — 3 cables

Asíncrono — sin reloj compartido. Misma velocidad de baudios en ambos lados. Bits de inicio, datos y parada.

  • TX — transmisor (→ RX del otro)
  • RX — receptor (← TX del otro)
  • GND — tierra común obligatoria
PWM — duty cycle

Señal digital que alterna ON/OFF a alta frecuencia. El porcentaje del tiempo en ON = duty cycle. Simula tensión media analógica.

  • 0% — siempre LOW (0V)
  • 50% — mitad ON, mitad OFF
  • 100% — siempre HIGH (3.3V)
  • Pines HW — 12, 13, 18, 19 en Raspi
04 — Sensores

Clasificación de sensores

Los sensores detectan cambios en el entorno y los convierten en señales eléctricas. Se clasifican de cuatro maneras distintas:

Por magnitud
Temperatura Presión Movimiento/Proximidad Luz/Ópticos Humedad Gas/Químicos Biosensores
Por fuente energía
Pasivos — no necesitan alimentación externa (PIR, termopar) Activos — emiten energía propia (ultrasonido, LiDAR, radar)
Por señal de salida
Analógicos — salida continua proporcional (LDR, termistor) Digitales — salida discreta binaria o bus (DHT22, DS18B20)
Por principio físico
Resistivos Capacitivos Inductivos Piezoeléctricos Ópticos

Sensores concretos para la Raspberry Pi

Temperatura

Sensores de temperatura

  • DS18B20 — Digital 1-Wire. Precisión ±0.05°C. Rango −55°C a +125°C. Pull-up 4.7kΩ. Varios en el mismo GPIO.
  • DHT11 — Temperatura + humedad. Rango 0–50°C. Económico, menos preciso.
  • DHT22 — Temperatura + humedad. ±0.5°C. Rango −40°C a +80°C. Superior al DHT11.
  • BMP280 — I2C/SPI. Temperatura + presión. Estaciones meteorológicas.
  • BME280 — I2C/SPI. Temperatura + presión + humedad. Todo en uno.
  • BME680 — Añade calidad del aire (VOC) al BME280.
1-WireI2CSPI
Movimiento / Distancia

Sensores de movimiento

  • HC-SR501 (PIR) — Infrarrojo pasivo. Detecta calor corporal. 3 pines (5V, GND, señal). Potenciómetros de ajuste. Salida digital.
  • HC-SR04 (Ultrasonido) — Activo. Emite ondas y mide eco. Rango 2–450 cm. 4 pines. Divisor de voltaje en entrada (5V→3.3V).
  • RCWL-0516 (Microondas) — Detecta movimiento a través de paredes finas. 3 pines. Mayor rango que PIR.

Tipos de detección PIR: infrarrojos pasivos (2 sensores piroeléctricos). Tipos ultrasonido: tiempo de vuelo del eco. Microondas: efecto Doppler.

Luz / Ópticos

Sensores de luz

  • LDR (Fotorresistencia) — Resistencia disminuye con más luz. Analógico → necesita ADC externo (MCP3008).
  • BH1750 — Digital I2C. Mide en lux. Fácil de programar con Python.
  • TSL2561 — I2C. Luz visible + infrarroja. Amplio rango dinámico.

Tipos ópticos: barrera (emisor y receptor separados), reflectivos (con catadióptrico), difusos (refleja el objeto).

Gas / Humo

Sensores de gas

  • Serie MQ — Semiconductores de óxido metálico (MOS). Detectan CO, gas, alcohol, humo. Analógicos → ADC externo (MCP3008).
  • BME680/BME688 — I2C. Temperatura + presión + humedad + VOC (calidad del aire).
  • SGP40 — I2C. VOC específico.

Tipos: electroquímicos, catalíticos, NDIR (infrarrojos), semiconductores MOS, fotoionización (PID), ultrasónicos.

Presión

Sensores de presión

Proceso: fuerza → deformación diafragma → cambio eléctrico → señal procesada.

  • BMP280 — I2C/SPI. Presión + temperatura.
  • BME280 — I2C/SPI. Presión + temperatura + humedad.
  • BMP390 — 24 bits. Mayor resolución para altitud.
  • Sensores industriales (salida analógica) → necesitan ADS1015/ADS1115 (ADC I2C)

Tecnologías: piezorresistivos, capacitivos, piezoeléctricos, strain gauges.

Principios físicos

Tipos por principio

  • Resistivos — resistencia cambia con magnitud. LDR, termistor (NTC/PTC), FSR, potenciómetro. → divisor de voltaje + ADC.
  • Capacitivos — detectan sin contacto. Metales, plásticos, líquidos, a través de paredes no metálicas.
  • Inductivos — solo metales. Campo EM. Corrientes parásitas atenúan oscilación. Industria.
  • Piezoeléctricos — fuerza → voltaje. Vibración, acelerómetros, micrófonos.
  • Ópticos — LED emisor + fotodiodo receptor. Barrera, reflectivo, difuso.

Convertidores ADC — imprescindibles para señales analógicas en Raspberry Pi

MCP3008

Resolución: 10 bits (0–1023). 8 canales. Interfaz SPI. El más usado en proyectos básicos. Ideal para LDR, potenciómetros, termistores.

ADS1115

Resolución: 16 bits. 4 canales (o 2 diferenciales). Interfaz I²C. PGA interno. Para señales muy pequeñas. Instrumentación y médico.

ADS1015

Resolución: 12 bits. 4 canales. Interfaz I²C. Intermedio entre MCP3008 y ADS1115. Más simple que el 1115.

05 — Actuadores

Actuadores — los "músculos" del IoT

El actuador convierte señal eléctrica/digital en acción física. Se clasifican por fuente de energía y por tipo de movimiento.

Eléctricos — los más comunes en IoT

Actuadores eléctricos

Convierten energía eléctrica en movimiento. Fácil integración con electrónica.

Ejemplos: LED, Relay, Servomotor (SG90), Motor DC, Motor paso a paso, resistencia calefactora, solenoides, displays.

Neumáticos — entornos industriales

Actuadores neumáticos

Utilizan aire comprimido. Fuerza y velocidad altas. Entornos industriales.

Ejemplos: Cilindros neumáticos, motores de giro neumáticos.

Hidráulicos — alta potencia

Actuadores hidráulicos

Líquido (aceite) a presión. Gran fuerza. Alta potencia.

Ejemplos: Cilindros hidráulicos, motores de giro hidráulicos.

LED — conexión a GPIO

Ánodo (pata larga) → GPIO pin. Cátodo (pata corta) → resistencia (330Ω) → GND.

GPIO 17 → [ánodo] LED [cátodo] → 330Ω → GND

LED RGB: ánodo o cátodo común, 3 resistencias (una por color R, G, B).

Servomotor — conexión PWM

Control preciso de posición mediante señal PWM. Tiene engranajes internos + potenciómetro de retroalimentación.

  • Marrón/Negro → GND
  • Rojo → 5V
  • Naranja/Amarillo → GPIO PWM (ej. GPIO17)

06 — Criterios de selección

Cómo elegir un sensor o actuador

Criterios ambientales

Temperatura

Rango de operación que supere las temperaturas extremas del entorno + margen de seguridad.

Humedad

Identificar HR min/max esperada. >80% HR puede degradar componentes. Evaluar recuperación de condensación.

Corrosión

Agentes corrosivos (ácidos, sales, agua de mar). Concentración + temperatura + tipo de exposición (continua vs. intermitente).

EMI / EMC

Identificar fuentes de ruido EM cercanas (motores, variadores). Inmunidad del dispositivo a interferencias. También sus propias emisiones.

Tamaño y Robustez

Compatible con el espacio disponible. Resistencia a golpes (estándares IK), vibraciones (frecuencia y amplitud), caídas.

Consumo de energía

Modo sleep (µA o nA), modo activo, picos de corriente. Protocolo inalámbrico: LoRaWAN/BLE (bajo) vs. Wi-Fi/4G (alto).

Criterios del sensor

Sensibilidad

Capacidad de detectar cambios pequeños en la magnitud medida y producir señal proporcional.

Rango

Intervalo mínimo–máximo que el sensor puede medir de forma fiable. Evitar saturación o falta de detección.

Precisión vs. Exactitud

Precisión: misma lectura en condiciones iguales (repetibilidad). Exactitud: qué tan cerca del valor real (rango de error).

Estabilidad y Linealidad

Estabilidad: mantiene sus características a lo largo del tiempo. Linealidad: relación constante entrada-salida; simplifica calibración.

criterios económicos — coste total de propiedad (TCO)
  • Precio de adquisición inicial + descuentos por volumen
  • Coste de instalación, cableado y configuración
  • Consumo energético a largo plazo
  • Mantenimiento, calibración y reparación
  • Disponibilidad de repuestos y soporte técnico
  • Tiempo de vida alineado con el proyecto
  • Longevidad del protocolo de comunicación (evitar 2G/3G)
  • Estandarización — preferir estándares abiertos a propietarios
